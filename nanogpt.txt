For **NanoGPT’s hosted API** (`nano-gpt.com`), you adjust generation settings by passing parameters in the JSON body of the request. It is mostly **OpenAI-compatible**, so the pattern is familiar.

## 1. Basic chat completion with temperature

```bash
curl -X POST https://nano-gpt.com/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "google/gemini-3-flash-preview",
    "messages": [
      {
        "role": "user",
        "content": "Explain Java interfaces simply."
      }
    ],
    "temperature": 0.3,
    "top_p": 0.9,
    "max_tokens": 800
  }'
```

NanoGPT says the `/api/v1/chat/completions` endpoint accepts sampling and decoding knobs, and omitted fields fall back to the routed provider/model defaults. ([docs.nano-gpt.com][1])

## 2. Common settings you can adjust

| Setting              | What it does                                               | Practical values                               |
| -------------------- | ---------------------------------------------------------- | ---------------------------------------------- |
| `temperature`        | Controls randomness/creativity                             | `0.0–0.3` factual, `0.5–0.8` balanced/creative |
| `top_p`              | Nucleus sampling; limits token choices by probability mass | Usually `0.8–1.0`                              |
| `top_k`              | Samples only from top-k likely tokens                      | Commonly `40`, if supported                    |
| `max_tokens`         | Maximum generated output length                            | e.g. `500`, `1000`, `4096`                     |
| `stop`               | Stop generation when sequence appears                      | e.g. `["###"]`                                 |
| `frequency_penalty`  | Reduces repeated wording                                   | `0.0–0.5`                                      |
| `presence_penalty`   | Encourages introducing new topics                          | `0.0–0.5`                                      |
| `repetition_penalty` | Provider-agnostic repeat reduction                         | Often around `1.05–1.2`                        |
| `seed`               | Attempts repeatable output where provider supports it      | e.g. `42`                                      |

NanoGPT also lists more advanced knobs like `min_p`, `tfs`, `typical_p`, `mirostat_mode`, `mirostat_tau`, `mirostat_eta`, `logit_bias`, and `logprobs`, but support depends on the underlying routed model/provider. ([docs.nano-gpt.com][1])

## 3. Context window setting: important gotcha

You usually **do not directly increase the model’s real context window** through a normal API parameter.

NanoGPT has a parameter called:

```json
"model_context_limit": 20000
```

But this only applies when **Context Memory** is enabled. Their docs say it affects the **memory compression step**, not the target model’s own context window. Values below `10,000` are clamped internally. ([docs.nano-gpt.com][1])

Example:

```bash
curl -X POST https://nano-gpt.com/api/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "memory: true" \
  -d '{
    "model": "google/gemini-3-flash-preview",
    "messages": [
      {
        "role": "user",
        "content": "Continue from our previous discussion."
      }
    ],
    "model_context_limit": 20000,
    "stream": false
  }'
```

So mentally:

```text
max_tokens = how long the answer can be
model context window = how much input the model can consider
model_context_limit = memory compression context size, not a magic bigger context window
```

That last one is the sneaky footgun.

## 4. Enable memory

NanoGPT supports long conversations through context memory. You can enable it by either appending `:memory` to the model name or sending the `memory: true` header. ([docs.nano-gpt.com][1])

```json
{
  "model": "openai/gpt-5.2:memory",
  "messages": [
    {
      "role": "user",
      "content": "Keep our previous discussion in mind and continue."
    }
  ]
}
```

Or with header:

```bash
-H "memory: true"
```

## 5. Reasoning / thinking settings

For reasoning-capable models, NanoGPT supports `reasoning_effort`:

```json
{
  "model": "anthropic/claude-opus-4.6",
  "messages": [
    {
      "role": "user",
      "content": "Explain quantum entanglement step by step."
    }
  ],
  "reasoning_effort": "high",
  "max_tokens": 4096
}
```

Supported effort values include `none`, `minimal`, `low`, `medium`, `high`, and `xhigh`. ([docs.nano-gpt.com][1])

You can also hide reasoning output:

```json
{
  "reasoning": {
    "exclude": true
  }
}
```

One small but important detail: hiding reasoning output does **not necessarily disable reasoning compute** if you still set a non-`none` reasoning effort. ([docs.nano-gpt.com][1])

## 6. Python example

```python
import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://nano-gpt.com/api/v1"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json",
}

payload = {
    "model": "google/gemini-3-flash-preview",
    "messages": [
        {"role": "system", "content": "You are a precise technical tutor."},
        {"role": "user", "content": "Explain Java HashMap in simple terms."}
    ],
    "temperature": 0.2,
    "top_p": 0.9,
    "max_tokens": 1000,
    "frequency_penalty": 0.2,
    "presence_penalty": 0.0,
    "seed": 42
}

response = requests.post(
    f"{BASE_URL}/chat/completions",
    headers=headers,
    json=payload
)

print(response.json())
```

## Recommended presets

For your Java docs / local-agent-style projects, I’d start with:

```json
{
  "temperature": 0.2,
  "top_p": 0.9,
  "max_tokens": 1200,
  "frequency_penalty": 0.1,
  "presence_penalty": 0.0
}
```

For brainstorming:

```json
{
  "temperature": 0.8,
  "top_p": 0.95,
  "max_tokens": 1500
}
```

For code generation:

```json
{
  "temperature": 0.1,
  "top_p": 0.9,
  "max_tokens": 2000
}
```

For extraction / JSON output:

```json
{
  "temperature": 0,
  "top_p": 1,
  "max_tokens": 1000
}
```

The main thing: **temperature, top_p, max_tokens, penalties, stop sequences, and reasoning effort are request parameters. Context window is mostly model-dependent. `model_context_limit` only matters for NanoGPT memory compression, not for magically expanding the model’s real input window.**
